## Summary

Adds the `azd tool` command group for managing external tool dependencies (CLI tools, VS Code extensions, npm packages, and azd extensions). This gives users a unified way to discover, install, upgrade, and check the status of tools that integrate with `azd`.

### Commands

| Command | Description |
|---|---|
| `azd tool list` | Show available tools and their installation status |
| `azd tool install [tools...]` | Install tools with dependency resolution and retry verification |
| `azd tool upgrade [tools...]` | Upgrade installed tools to latest versions |
| `azd tool check` | Check for available tool updates |
| `azd tool show <tool>` | Show detailed information about a specific tool |

## Changes

### Command Layer (`cli/azd/cmd/`)

- **`cmd/tool.go`** — Registers the `azd tool` command group with subcommands: `list`, `install`, `upgrade`, `check`, `show`

### Core Package (`cli/azd/pkg/tool/`)

- **`pkg/tool/detector.go`** — Concurrent tool detection using `sync.WaitGroup` with spinner UX
- **`pkg/tool/installer.go`** — Installation orchestration with dependency resolution, retry with exponential backoff, and post-install verification
- **`pkg/tool/manager.go`** — Central manager coordinating detection, installation, and upgrade workflows
- **`pkg/tool/manifest.go`** — Tool manifest definitions and parsing; supports CLI tools, VS Code extensions, npm packages, and azd extensions
- **`pkg/tool/platform.go`** — Cross-platform support (Windows, macOS, Linux) with platform-specific install strategies and checksum validation for direct downloads
- **`pkg/tool/update_checker.go`** — Background update check logic

### Middleware (`cli/azd/cmd/middleware/`)

- **`middleware/tool_first_run.go`** — First-run middleware that prompts new users to install recommended tools on initial `azd` usage
- **`middleware/tool_update_check.go`** — Background middleware that notifies users when tool updates are available

## Testing

- CI/non-interactive environment detection ensures no interactive prompts in automated pipelines
